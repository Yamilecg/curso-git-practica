# curso-git-practica
Repositorio de práctica para el curso de Git y GitHub

## Commits Convencionales

Todos los commits en este repositorio deben seguir las [especificaciones de commits convencionales](https://www.conventionalcommits.org/en/v1.0.0/#specification). Esto ayuda a mantener un historial de commits claro y estructurado.

**Formato de los Commits:**
- **Tipo:** (feat, fix, docs, style, refactor, perf, test, chore)
- **Descripción:** Breve descripción del cambio (máximo 50 caracteres).
- **Cuerpo (opcional):** Descripción más detallada del cambio.
- **Pie de página (opcional):** Referencias a tickets o problemas.
